# Architecture

## Design Overview

The bot is structured as two cooperating processes: a FastAPI HTTP/WebSocket server that runs in a daemon thread, and a sequential scenario loop that runs on the main thread. When `main.py` starts, it boots the server, opens an ngrok HTTPS tunnel, and then iterates through scenarios one at a time. For each scenario, it calls `set_scenario_context()` to write the active persona into a module-level variable, then uses the Twilio SDK to dial the target number with the ngrok URL as the webhook. When Twilio connects the call and the target answers, it POSTs to `/answer`, which returns TwiML that opens a WebSocket back to `/stream`. All subsequent audio flows through that WebSocket: Twilio sends μ-law encoded audio from the agent in 20 ms chunks, and we send μ-law audio back as the patient. This architecture keeps all logic in a single Python process — no Redis, no queues, no external coordination.

## Key Technical Decisions

**Audio codec in pure numpy.** Python's `audioop` module was deprecated in 3.11 and removed in 3.13. Rather than adding a compatibility shim (`audioop-lts`), the ITU G.711 μ-law encode and decode are implemented directly with vectorised numpy operations. The decode formula (`~byte → split sign/exp/mantissa → reconstruct linear magnitude`) and encode formula (sign extraction → BIAS add → exponent lookup table → mantissa shift → bit inversion) both match the G.711 specification and have been verified against known silence and maximum-amplitude bytes. Resampling from OpenAI TTS's native 24 kHz output to Twilio's required 8 kHz is handled by pydub's `set_frame_rate()`, which applies an anti-aliased FIR filter — simple decimation (every third sample) would alias and sound distorted. **End-of-turn detection** uses a 800 ms RMS silence threshold on the accumulated PCM buffer; the `is_processing` flag prevents overlapping Whisper/GPT/TTS tasks if the agent sends rapid bursts. The `[[END_CONVERSATION]]` sentinel in the patient's GPT response allows the persona to signal conversation completion without a separate channel — it is stripped before TTS so the token never reaches the audio stream. Post-call QA uses a low-temperature (0.2) GPT-4o-mini pass with a strict output schema (`BUG:` / `SEVERITY:` / `TIMESTAMP:` / `EXPECTED:` / `ACTUAL:` / `---`) that is deterministically parseable with simple string splitting, avoiding the fragility of asking the model to emit JSON.
